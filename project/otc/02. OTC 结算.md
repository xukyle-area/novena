## 一、订单生命周期状态流转

```
PENDING_NEW → NEW → CONFIRMED → FILLED → SETTLED
                  ↘ PENDING_CANCEL → CANCELED / PARTIAL_SETTLED
```

状态定义（见 OtcOrderStatus.java）：
- `PENDING_NEW`: 订单创建，等待资产锁定确认
- `NEW`: 资产锁定成功，已向LP发送订单
- `CONFIRMED`: LP确认成交
- `FILLED`: 进入结算流程（结算准备完成）
- `SETTLED`: 结算完成
- `PARTIAL_FILLED / PARTIAL_SETTLED`: 部分结算

---

## 二、资产锁定时间节点

### 1. 用户资产锁定
**时机**: 用户下单时立即锁定

锁定内容：
- **锁定对象**: 用户账户的 `source` 资产（如买入BTC则锁定USDT）
- **锁定类型**: `LOCKED_OTC`
- **锁定金额**: 订单总金额

### 2. LP资产锁定
**时机**: 结算准备阶段（`prepareCombinationSettle` 或 `preparePartialSettle`）

锁定规则：
- 只锁定LP需要**净支出**的币种（`amount < 0`）
- 锁定前检查LP可用余额是否充足

---

## 三、资产解锁时间节点

### 1. 订单失败时解锁
**时机**:  定时任务中检测到锁定失败

### 2. 订单取消时解锁
**时机**:  定时任务处理状态订单

### 3. 结算完成时解锁
**时机**: 结算任务执行时，作为账本交易的一部分

---

## 四、资产划转时间节点

### 全量结算 (`combinationSettle`)
**时机**: 定时任务 `OtcOrderCombinationSettleTask` 检测到LP资产锁定成功后执行

划转步骤（单笔结算包含6个账本交易）：

| 步骤 | 操作               | 账户       | 币种      | 金额变化        |
| ---- | ------------------ | ---------- | --------- | --------------- |
| 1    | 解锁用户锁定资产   | 用户       | source    | 锁定值→剩余值   |
| 2    | 扣除用户source资产 | 用户       | source    | -sourceAmount   |
| 3    | 增加用户target资产 | 用户       | target    | +targetAmount   |
| 4    | 增加LP的source收入 | LP         | source    | +lpTargetAmount |
| 5    | 解锁LP锁定资产 | LP         | target    | -lpSourceAmount |
| 6    | 扣除LP的target支出 | LP         | target    | -lpSourceAmount |
| 7    | 差价打入markup账户 | Markup账户 | quote币种 | +markupAmount   |

### 部分结算 (`partialSettle`)
**时机**: 定时任务，当LP资产不足以全量结算时

- 按LP当前可用余额计算可结算数量
- 最小结算金额: `PARTIAL_SETTLE_MIN_AMOUNT`
- 订单状态变为 `PARTIAL_FILLED`，直到完全结算变为 `SETTLED`

## 五、完整时序流程图

```
用户下单 ──────────────────────────────────────────────────────────────────────
    │
    ▼ [立即] 锁定用户source资产 (LOCKED_OTC)
    │
    ▼ 状态: PENDING_NEW
    │
    ├──▶ [定时任务] 检查锁定成功 ──▶ 状态: NEW ──▶ 向LP发送订单
    │
    └──▶ [定时任务] 锁定失败 ──▶ 解锁资产 ──▶ 状态: FAILED

LP成交回调 ─────────────────────────────────────────────────────────────────────
    │
    ▼ 状态: CONFIRMED (等待结算)
    │
    ▼ [系统/人工] 准备结算 (prepareCombinationSettle)
    │
    ├──▶ 锁定LP需要支出的资产
    │
    ▼ 状态: FILLED

结算任务 ───────────────────────────────────────────────────────────────────────
    │
    ▼ [检查] LP资产锁定成功?
    │
    ├──▶ 成功: 执行账本划转 (combinationSettle)
    │         1. 解锁用户锁定资产
    │         2. 用户source → LP
    │         3. LP target → 用户
    │         4. 差价 → Markup账户
    │         5. 解锁LP锁定资产
    │
    │    ▼ 状态: SETTLED
    │
    └──▶ 失败: 重试锁定LP资产

取消订单 ───────────────────────────────────────────────────────────────────────
    │
    ▼ 状态: PENDING_CANCEL
    │
    ▼ [定时任务] cancelOtcOrder
    │
    ├──▶ 解锁用户资产
    │
    ├──▶ 无结算记录: 状态→CANCELED
    │
    └──▶ 有部分结算: 状态→PARTIAL_SETTLED
```
