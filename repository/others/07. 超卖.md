
超卖（Overselling）一般出现在高并发场景下，例如电商秒杀、抢票、交易撮合等。本质问题是：**多个请求同时扣减库存，导致库存被重复扣减，出现负数或超过真实库存。**

常见解决方案可以按**复杂度和可靠性**分为几类：

---

# 一、数据库层解决（最简单）

### 1. 乐观锁（推荐）

通过 **version 或库存条件更新**控制并发。

**SQL 示例**

```sql
UPDATE product 
SET stock = stock - 1
WHERE id = 1001 AND stock > 0;
```

判断影响行数：

* `rows = 1` → 扣减成功
* `rows = 0` → 库存不足

优点：

* 实现简单
* 不需要额外组件
* 适合中等并发

缺点：

* 高并发时数据库压力大

---

### 2. version 乐观锁

表结构：

```sql
id
stock
version
```

更新：

```sql
UPDATE product
SET stock = stock - 1,
    version = version + 1
WHERE id = 1001
AND version = 10;
```

失败则重试。

优点：

* 数据一致性强

缺点：

* 高并发冲突严重

---

# 二、Redis 原子扣减（高并发常用）

核心思路：

**库存放在 Redis，用原子操作扣减**

Redis：

```
stock:1001 = 1000
```

扣减：

```bash
DECR stock:1001
```

或者：

```lua
if redis.call("get",KEYS[1]) > 0 then
    return redis.call("decr",KEYS[1])
else
    return -1
end
```

优点：

* Redis 单线程
* 原子操作
* 性能极高

缺点：

* 需要处理 Redis 与 DB 一致性

---

# 三、Redis + Lua（秒杀标准方案）

Lua脚本：

```lua
local stock = redis.call('get', KEYS[1])

if tonumber(stock) <= 0 then
    return -1
end

redis.call('decr', KEYS[1])
return 1
```

优点：

* **原子性**
* **无并发问题**
* **秒杀系统标准做法**

---

# 四、消息队列削峰（大型系统）

流程：

```
用户请求
   ↓
Redis 扣库存
   ↓
写入 MQ
   ↓
订单服务消费 MQ
   ↓
创建订单
   ↓
写数据库
```

架构：

```
用户
 ↓
API
 ↓
Redis库存
 ↓
MQ
 ↓
订单服务
 ↓
DB
```

优点：

* 抗 **10w+ QPS**
* 削峰填谷

缺点：

* 系统复杂

---

# 五、分布式锁（不推荐）

例如：

* Redis Lock
* Zookeeper Lock

流程：

```
获取锁
 ↓
扣库存
 ↓
释放锁
```

缺点：

* 性能差
* 锁竞争严重

一般不用来解决超卖。

---

# 六、预扣库存（交易系统常用）

例如交易所、票务系统：

```
库存 = 可用库存 + 锁定库存
```

流程：

```
下单
  ↓
锁定库存
  ↓
支付成功
  ↓
正式扣减
```

类似：

* 机票
* 火车票
* 币交易订单

---

# 七、终极秒杀架构（大厂方案）

完整流程：

```
用户请求
   ↓
网关限流
   ↓
Redis库存预扣
   ↓
MQ削峰
   ↓
订单服务
   ↓
数据库扣减
   ↓
异步对账
```

特点：

* Redis 控并发
* MQ 削峰
* DB 最终一致

---

# 八、最推荐的方案（实践）

高并发系统一般这样做：

**Redis + Lua + MQ**

流程：

```
1 用户请求
2 Redis Lua 扣库存
3 成功 → 写 MQ
4 订单服务消费 MQ
5 创建订单
6 DB 扣库存
```

---

# 九、一句话总结

解决超卖的核心是：

**保证库存扣减的原子性**

常见手段：

| 方案          | 并发能力 | 推荐  |
| ----------- | ---- | --- |
| 数据库库存条件更新   | 中    | √   |
| Redis原子扣减   | 高    | √√  |
| Redis + Lua | 很高   | √√√ |
| MQ削峰        | 极高   | √√√ |
| 分布式锁        | 低    | ×   |
