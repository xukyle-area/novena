Redis 的 **`zset`（有序集合）** 底层设计非常巧妙，它同时满足 **快速按 member 查找** 和 **按 score 排序范围查询** 的需求。可以分成两部分来理解：

---

## 1️⃣ 核心要求

`zset` 需要满足：

1. **按 member 查找 O(1)** → 快速判断 member 是否存在或获取它的 score
2. **按 score 排序 O(log N)** → 支持范围查询、排名查询、top N 等操作

单独使用哈希表或跳表都无法同时满足，所以 zset 使用 **哈希表 + 跳表** 的组合。

---

## 2️⃣ 组成结构

### （1）哈希表（dict）

* key：member 字符串
* value：score（double）
* 作用：

  * 快速查找 member 是否存在
  * 快速获取 score
* 时间复杂度：O(1)

### （2）跳表（skip list）

* 节点存储：

  * member 字符串
  * score（double）
* 节点按 **score 升序排列**

  * score 相同 → 按 member 的字典序排列
* 作用：

  * 快速按 score 排序
  * 快速范围查询：ZRANGEBYSCORE、ZRANGE、ZREVRANGE
* 时间复杂度：

  * 查找/插入/删除：O(log N)
* 每个节点通常包含多层指针，用于加快跳跃查找

---

## 3️⃣ 如何组合使用

* **哈希表** → 用于 O(1) 定位 member
* **跳表** → 用于按 score 排序和范围操作
* 插入新 member：

  1. 先在哈希表查找 member 是否存在
  2. 如果不存在 → 插入哈希表
  3. 再插入跳表按 score 排序
* 更新 member score：

  1. 哈希表找到旧 score
  2. 跳表删除旧节点
  3. 跳表插入新节点

> 因此，zset 能同时做到 **快速查找 + 有序操作**，效率很高。

---

## 4️⃣ 内存示意

```
zset key: leaderboard
----------------------------------------------------
| Hash Table             | Skip List (by score)      |
|-----------------------|--------------------------|
| "alice" -> 100        | 100 → "alice"            |
| "bob"   -> 95         | 100 → "bob"              |
| "carol" -> 120        | 120 → "carol"            |
----------------------------------------------------
```

* 哈希表定位 member
* 跳表支持按 score 排序或范围操作

---

## 5️⃣ 特点总结

| 优点          | 描述                     |
| ----------- | ---------------------- |
| 查找 member   | O(1)（哈希表）              |
| 插入/删除       | O(log N)（跳表）           |
| 排序查询        | O(log N + M)（M = 返回数量） |
| 支持 score 相同 | 节点按 member 字典序排序       |

---

💡 **核心理解**：
`zset = 哈希表 + 跳表`

* 哈希表保证 **唯一性 + 快速查找**
* 跳表保证 **按 score 排序 + 范围查询**
